But before that, static factory method:-
A static factory method is a static method that returns an instance of a class, instead of using a constructor directly (new).
Simple definition
An alternative to constructors for creating objects, with more control and flexibility.

User user = new User("Pawan", 30);

User user = User.of("Pawan", 30);

Why use a static factory method?
1Ô∏è‚É£ Meaningful names (unlike constructors)
Constructors must have the class name. Static methods can explain intent
LocalDate.now()
LocalDate.of(2025, 12, 21)
Much clearer than multiple constructors.

2Ô∏è‚É£ Can control object creation
You can:
Return an existing object
Cache objects
Return a subclass
public static Boolean valueOf(boolean b) {
    return b ? Boolean.TRUE : Boolean.FALSE;
}

3Ô∏è‚É£ Can return different implementations
Very useful in backend / system design.
public static List<String> createList() {
    return new ArrayList<>();
}
Caller doesn‚Äôt care about the actual implementation.

4Ô∏è‚É£ Avoid constructor overloading hell
Instead of:

new User("Pawan", 30, "India")
new User("Pawan", 30, "India", true)
You get:

5Ô∏è‚É£ Can validate before creating object

    if (age < 0) {
        throw new IllegalArgumentException("Age cannot be negative");
    }
    return new User(name, age);
